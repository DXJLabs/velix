# Messaging

Messaging happens inside a Deal Room.

Users use messages to discuss terms, clarify payment context, coordinate proof references, and keep the conversation attached to the deal.

## User Flow

1. Open a channel.
2. Review the counterparty and deal status.
3. Type a message.
4. Confirm the privacy mode when prompted.
5. Send the message.
6. Review the message in channel history.

## User Expectations

- Messages belong to a specific channel.
- Sensitive deal context should not be public by default.
- Messages should stay connected to offers, memos, and escrow activity.
