# Negotiation

Negotiation keeps offers and decisions inside the channel.

Users should not need a separate chat thread, spreadsheet, or offchain note to understand the state of a deal.

## User Flow

1. Open a Deal Room.
2. Review existing terms and status.
3. Create an offer or counter-offer.
4. Wait for the counterparty to accept, reject, or counter.
5. Continue until terms are accepted or abandoned.
6. Move accepted terms toward payment memo and escrow actions.

## User Expectations

- Offers should be tied to the current channel.
- The latest offer state should be easy to identify.
- Accepted terms should lead naturally into escrow or payment.
