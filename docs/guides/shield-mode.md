# Shield Mode

Shield mode is the privacy-first user choice.

Users choose Shield when the action should use the strongest available VEIL privacy path for metadata-resistant deal coordination.

## When To Use Shield

- Sensitive payment context.
- Private negotiation.
- Escrow coordination where counterparty or timing metadata matters.
- Proof references that should not be broadly exposed.

## User Flow

1. Choose Shield in the action view or privacy settings.
2. Review the action summary.
3. Confirm only when the product reports Shield support for the environment.
4. Send the action.
5. Review the result in channel activity.

## User Expectations

- Shield should be clearly labeled.
- If Shield is unavailable, the product should fail clearly instead of silently falling back.
- Users should understand that Shield is for privacy-sensitive actions.
