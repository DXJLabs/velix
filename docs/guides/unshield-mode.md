# Unshield Mode

Unshield mode is an explicit visible path.

Users choose Unshield for testing, direct public/testnet activity, or flows where privacy is not required.

## When To Use Unshield

- Testnet demos.
- Public or non-sensitive actions.
- Debugging a channel flow.
- Cases where the user intentionally wants a direct transaction path.

## User Flow

1. Choose Unshield in the action view.
2. Review the visibility warning.
3. Confirm the action.
4. Review the result in channel activity.

## User Expectations

- Unshield should never be hidden or automatic for sensitive actions.
- Users should understand they are choosing a more visible path.
- The action should still stay organized inside the Deal Room.
