# Core Features

## Deal Room

A channel-centered workspace for messages, payment context, offers, proofs, and escrow status.

## Messaging

Users exchange private channel messages without making the conversation public by default.

## Payment Memo

Users attach structured payment context before sending or reviewing payment actions.

## Escrow

Users coordinate buyer/seller milestones: create, confirm deposits, activate, release, settle, or cancel.

## Negotiation

Users propose offers, counter-offers, acceptances, and rejections inside the same channel where settlement context lives.

## Shield Mode

Users select a privacy-preserving mode for sensitive actions when the product environment supports it.

## Unshield Mode

Users can intentionally use a direct visible path for testing, explicit public actions, or flows where privacy is not required.

## Wallet

Users connect a Starknet wallet, review readiness, and check asset status.

## Activity

Users see pending actions and recent history across channels.

## Settings

Users manage privacy defaults, memo requirements, notifications, sessions, and wallet controls.
