# Problem

Onchain payments are transparent, but deal context is often sensitive.

Users need to coordinate terms, memos, proof references, and escrow state. That context can reveal relationships, pricing, timing, counterparties, and commercial intent.

## Current User Pain

- Public transactions expose too much metadata.
- Payment memos are often disconnected from the payment.
- Negotiation happens in private chat apps that are separate from settlement.
- Escrow status is hard to understand without external coordination.
- Partners and reviewers cannot easily see the product value when documentation starts with SDK internals.

## Why This Matters

Privacy is not only about hiding balances. In real workflows, metadata is often the product risk.

VEIL focuses on metadata-resistant communication around payments and escrow.
